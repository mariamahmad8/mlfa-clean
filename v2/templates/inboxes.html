Setup requirements — read before adding a mailbox

Before an inbox will work, Microsoft-side setup must be done:

  1. The mailbox must exist in your Microsoft 365 tenant (e.g. give@mlfa.org).
  2. The Azure App Registration for this system needs application permissions: Mail.ReadWrite, Mail.Send, MailboxSettings.ReadWrite. Admin consent must be granted.
  3. If IT set up an Application Access Policy (scoping which mailboxes the app can touch), the new mailbox must be added to the whitelist. Ask IT to run: 
    Add-DistributionGroupMember -Identity "PairAppAccessGroup" -Member "give@mlfa.org"
    If IT did NOT set up scoped access, the app can already reach any mailbox — nothing to do here.
  4. Once Microsoft-side is done, add the inbox below. The worker will pick it up on the next poll cycle (roughly 10 seconds).

Cloning: if a similar inbox already exists, use "Clone from" in the Add form to copy over all category rules, recipients, and templates. Much faster than starting from scratch.
